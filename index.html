🛠️ Manutenção de dados

Limpar entrevistador do histórico de etapas. Remove o nome do entrevistador gravado em todas as etapas (Acompanhar Espécies) de todos os domicílios, voltando o campo para "Não informado". O resultado de cada etapa e o mês/ano são preservados — só o nome do entrevistador é apagado. Use isso uma única vez para corrigir o espelhamento indevido feito anteriormente; depois disso, reimporte os CSVs/SIGC normalmente.

Remover domicílios duplicados sem período. Antes da divisão por períodos (Jan/Abr/Jul/Out, Fev/Mai/Ago/Nov, Mar/Jun/Set/Dez), os domicílios eram salvos sem o campo periodo. Ao reimportar depois disso, o sistema criou um novo documento com período definido para cada domicílio — e o documento antigo (sem período) ficou no banco, sendo contado em dobro no Dashboard. Esta ferramenta encontra domicílios com o mesmo Controle + Número que existem tanto SEM período quanto COM período; antes de excluir a versão antiga, ela copia automaticamente para a versão atual qualquer dado importante que só exista na antiga (coordenadas do KML, histórico de etapas, dados do SIGC etc.), evitando perda de informação.

Importar histórico de entrevistadores (.xlsx). Carrega a planilha com colunas MÊS, UPA, CONTROLE, DOM., ENTREVISTADOR 1…5 e preenche o histórico de etapas de cada domicílio — mesclando com o que já existe (não apaga dados do SIGC).

Sincronizar status pelo histórico de espécies. Verifica todos os domicílios onde o historico_especies da etapa atual indica um resultado final (Uso Ocasional, Vago, Realizada, etc.) mas o status_atual está desatualizado. Corrige apenas os casos onde o histórico aponta um status terminal diferente do registrado.

Limpar todas as solicitações. Remove o campo solicitacoes e tipo_solicitacao de todos os domicílios. Domicílios cujo status seja supervisao ou carta_solicitada voltam para nao_visitado. Use para apagar solicitações de teste antes de começar o uso real.
